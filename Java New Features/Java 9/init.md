JDK 9

The goal of this Project is to produce an open-source reference implementation of the Java SE 9 Platform defined by JSR 379 in the Java Community Process.
이 프로젝트의 목표는 Java Community Process에서 JSR 379에 정의 된 Java SE 9 Platform의 오픈 소스 참조 구현을 생성하는 것입니다.

The schedule and features of this release are proposed and tracked via the JEP Process, as amended by the JEP 2.0 proposal.
이 릴리스의 일정과 기능은 JEP 2.0 제안에 의해 개정 된 JEP 프로세스를 통해 제안되고 추적됩니다.

Status

We are now in the second phase of the rampdown process, in which we aim to fix just the bugs that must be fixed in order to ensure a successful release and understand why we're not going to fix some bugs that perhaps ought to be fixed. Please see the RDP 2 page for details.
우리는 이제 램프 다운 프로세스의 두 번째 단계에 있습니다. 성공적인 릴리스를 보장하고 수정해야 할 버그를 수정하지 않는 이유를 이해하기 위해 수정해야하는 버그를 수정하는 것을 목표로합니다 . 자세한 내용은 RDP 2 페이지를 참조하십시오.

The overall feature set is, at this point, frozen. It is highly unlikely that any further JEPs will be targeted to the release. Low-risk enhancements that add small bits of missing functionality or improve usability may be approved, especially when justified by developer feedback, but the bar is now extremely high. API or other specification changes made by a JSR Expert Group are critical by definition, and will be approved. You can request approval for an enhancement via the existing FC extension-request process but, as of RDP 2, only the JDK 9 Project Lead or a delegate, in the case of absence, may approve such requests.
전체 기능 세트는 이 시점에서 고정됩니다. 더 이상의 JEP가 릴리스 대상이 될 가능성은 거의 없습니다. 누락 된 기능을 추가하거나 사용성을 향상시키는 위험성이 낮은 기능 향상은 승인 될 수 있습니다. 특히 개발자 의견에 의해 정당화 될 때 승인 될 수 있지만 바는 현재 매우 높습니다. JSR 전문가 그룹이 만든 API 또는 기타 사양 변경은 정의 상 중요하며 승인 될 것입니다. 기존 FC 확장 요청 프로세스를 통해 향상에 대한 승인을 요청할 수 있지만 RDP 2의 경우 JDK 9 프로젝트 리드 또는 부재자의 경우 대표 만이 그러한 요청을 승인 할 수 있습니다.

Milestone definitions

The milestone definitions for JDK 9 are the same as those for JDK 8, with the addition of:

Feature Extension Complete — The date by which JEPs and small enhancements that have been granted extensions via the FC extension-request process must be integrated into the master forest.

Initial Release Candidate — The date on which the first release candidate will be built and submitted for testing.
